# mvts-forecast

A **leakage-free multivariate-transformer forecast benchmark** with an honest
NULL headline.

> **Honest headline:** on noisy daily **returns**, a PatchTST-style encoder and a
> simplified interpretable variable-selection transformer (and an LSTM) do **NOT**
> reliably beat a naive random-walk baseline out-of-sample on directional accuracy
> or risk-adjusted PnL after costs — Diebold-Mariano insignificant, Deflated
> Sharpe ~ 0. The deliverable is the rigorous, leakage-free comparison, **not** a
> profit claim.

The shipped default runs the comparison on a **synthetic multivariate panel** (a
weak common factor + dominant idiosyncratic noise + mild autocorrelation, seeded —
no API keys) plus committed, offline-trained **ONNX** deep models served via
onnxruntime (the serve container has **no torch**). Real data flows through the
yfinance → Stooq + FRED-CSV offline CLI path.

## Why this exists

This is the rigorous counterweight to the "predict the stock price with an LSTM
and report a 0.99 R²" genre. That R² is a unit-root artifact of regressing a
trended price level on its own lag — **not** forecasting skill. Here:

- the target is the next-step **RETURN**, never the price level;
- there is **no price-level R²** anywhere;
- RevIN / instance-norm is computed from the **input window only**, the
  standardizer is fitted on the **train fold only**, and the walk-forward is
  **purged (≥ `look_back`) + embargoed** so no window straddles a split;
- the encoder input never contains the target's same-step transform;
- the verdict `deep_beats_naive` is a **pure function** of the evidence and reads
  `False` unless a deep model beats naive with a **DM-significant** margin **and** a
  **positive Deflated Sharpe**.

## Reproduce

```bash
# 1. Lean (torch-free) install — the serve / baseline path.
uv venv && uv pip install -e ".[data,serve,viz,dev]"

# 2. Run the live, torch-free baselines (naive + ARIMA) on the synthetic panel.
mvts-forecast forecast --data-source synthetic --target SPY --horizon 1 --seed 7

# 3. Run the full deep-vs-naive comparison with the PURE verdict.
#    Deep models are served from the committed ONNX artifacts (onnxruntime, NO torch).
mvts-forecast compare --data-source synthetic --target SPY --horizon 1 --seed 7

# 4. Verify every correctness gate (ruff + strict mypy + coverage >= 85).
ruff check src tests && mypy src
pytest -q                       # 315 passed, 1 skipped; coverage ~93%
pytest -q -m parity             # ONNX<->torch 1e-4 (needs [train]), DSR 1e-10, DM
pytest -q -m regression         # the random-walk anti-leakage lock

# 5. Retrain offline and re-export the ONNX artifacts (the ONLY torch step).
uv pip install -e ".[train]"
mvts-forecast train --data-source synthetic --seed 7
#   real data: mvts-forecast train --data-source yfinance --basket SPY,TLT,GLD
```

The committed `metrics.json` is regenerated byte-for-byte from the same
`seed=7` `RunManifest`; the parity test certifies the ONNX graph the container
serves is the model the `[train]` path produced.

## Results — the honest NULL, measured

The shipped default (synthetic panel, `target="SPY"`, `horizon=1`, `lookback=60`,
`seed=7`, full-grid `n_effective_trials=3`) returns the precomputed
`metrics.json` for the deep models plus the live naive/ARIMA baselines. The
**best model is `naive`** and **`deep_beats_naive = false`**:

| Model | Return-space RMSE | Directional accuracy | DM p-value vs. naive | Net-PnL Sharpe | Deflated Sharpe |
|-------|------------------:|---------------------:|---------------------:|---------------:|----------------:|
| **`naive`** (random walk) | **0.01099** | — (reference) | 1.00 | 0.000 | 0.197 |
| `lstm` | 0.01294 | 0.504 | 4.5e-19 | -0.020 | 0.058 |
| `transformer` | 0.05158 | 0.507 | 5.0e-145 | 0.088 | 0.990 |
| `patchtst` | 0.35585 | 0.502 | 1.7e-175 | 0.042 | 0.750 |

Read it carefully — the tiny DM p-values do **not** mean the deep models win.
The naive forecast has the **lowest** RMSE; every deep model's loss differential
is signed *against* it, so the DM test is significant **in naive's favour**. The
verdict gate (DM-significant **and** in the model's favour **and** DSR > 0) is
therefore never cleared:

- **Directional accuracy** is a coin flip for every deep model (0.502–0.507);
  the binomial test does not reject 0.5.
- **Deflated Sharpe** is ~0 once deflated against the `n_effective_trials = 3`
  multiplicity-inflated benchmark — no risk-adjusted edge survives selection.
- **No price-level R²** appears anywhere; all skill is judged in return space
  (see [ADR-0005](docs/decisions/0005-no-price-level-r2.md)).

> **Verdict:** `deep_beats_naive = false`. On noisy daily returns, PatchTST, the
> interpretable transformer, and the LSTM do **not** reliably beat a naive
> random walk out-of-sample. The deliverable is this leakage-free comparison.

### Validation (correctness gates, not performance)

| Gate | Check | Tolerance |
|------|-------|-----------|
| ONNX ↔ torch parity | each exported deep model's ONNX forward matches its torch forward | `1e-4` (rtol/atol) |
| Deflated Sharpe parity | `evaluation/dsr.py` vs. the reused HRP reference | `1e-10` |
| Diebold-Mariano | statistic/p-value vs. closed-form + Newey-West HAC oracle | exact / `1e-10` |
| Future-perturbation invariance | RevIN + windowing forecast at `t` is unchanged when rows at `t+1..` are altered | exact (Hypothesis) |
| No-target-in-features | encoder input never contains the target's same-step transform | asserted |
| Random-walk anti-leakage | on a pure random walk the deep models do **not** beat naive (`deep_beats_naive=false`) | regression-locked |

Reproduce the numbers and gates locally with the [Reproduce](#reproduce) block.

## Limitations

- **Static basket / survivorship — N/A here, but flagged:** the default basket is
  static and synthetic, so survivorship bias does **not** apply to the shipped
  result. A real-data run on a *fixed* listed universe **would** be exposed to it
  (delisted names silently dropped). The upgrade path is a **point-in-time (PIT)
  universe** — reconstruct membership as of each rebalance date so the backtest
  only ever sees what was investable then; the windowing/walk-forward layer is
  already PIT-ready, only the data loader needs the PIT membership table.
- **Synthetic-trained default:** the shipped ONNX models are trained on the
  synthetic panel, where the next-step return is dominated by unforecastable
  noise — so the honest NULL holds *by construction*. The numbers characterize a
  faithful, leakage-free pipeline on a known-null DGP, not a claim about any real
  asset. Retrain on real data via `mvts-forecast train --data-source yfinance`.
- **Macro release-date lags:** real macro features are lagged to their **RELEASE**
  dates (not the reference period they describe) to avoid look-ahead; the lag is a
  conservative proxy and a deliberately blunt one — a per-series PIT release
  calendar would sharpen it.
- **Single-seed shipped default:** the committed `metrics.json` is one
  `seed=7` run; the conclusion (a coin-flip directional accuracy and a deflated
  Sharpe of ~0) is stable across seeds by construction, but the README table
  reports the one reproducible default rather than a seed average.

## Design and decisions

The architecture, the data flow through one walk-forward window, the leakage
invariants, and the testing strategy are documented in
[`docs/DESIGN.md`](docs/DESIGN.md). The contested choices each have a numbered
ADR in [`docs/decisions/`](docs/decisions/):

- [ADR-0001](docs/decisions/0001-revin-input-window-only.md) — RevIN computed from the input window only.
- [ADR-0002](docs/decisions/0002-purge-embargo-walkforward.md) — purge (≥ `look_back`) + embargo walk-forward.
- [ADR-0003](docs/decisions/0003-onnx-serve-no-torch.md) — serve via ONNX / onnxruntime; torch is train-only.
- [ADR-0004](docs/decisions/0004-honest-null-vs-naive.md) — the pure `deep_beats_naive` verdict.
- [ADR-0005](docs/decisions/0005-no-price-level-r2.md) — never report a price-level R².
